# Messages Module - LangChain Message Types and Usage

This module demonstrates fundamental concepts for working with messages in LangChain. Messages are the core building blocks for interacting with language models, representing both input and output with full context and metadata.

## Prerequisites

- **Ollama**: Must be installed and running locally
- **Required Models**: 
  - `deepseek-r1:latest` (for example 1)
  - `qwen3.5:latest` (for examples 2 and 3)
  - `deepseek-coder:6.7b` (for example 4)

Install models with:
```bash
ollama pull deepseek-r1:latest
ollama pull qwen3.5:latest
ollama pull deepseek-coder:6.7b
```

Start Ollama service before running any examples:
```bash
ollama serve
```

## Message Fundamentals

Messages in LangChain contain three core elements:

| Element | Description |
|---------|-------------|
| **Role** | Identifies the message type (system, user, assistant) |
| **Content** | Actual message content (text, images, audio, documents, etc.) |
| **Metadata** | Optional fields (response info, message IDs, token usage, etc.) |

## Message Types

### SystemMessage
- **Purpose**: Initial instructions that prime the model's behavior
- **Use Case**: Define role, tone, guidelines, and behavioral rules
- **Best Practice**: Be specific and detailed for consistent model behavior

### HumanMessage
- **Purpose**: Represents user input and interactions
- **Use Case**: User queries, prompts, and requests
- **Supports**: Text, images, audio, and multimodal content

### AIMessage
- **Purpose**: Responses generated by the model
- **Contains**: Text content, tool calls, metadata, token usage information
- **Returned by**: All model invocations
- **Can be created manually**: For simulating conversation history

### ToolMessage
- **Purpose**: Represents results of tool execution
- **Use Case**: Tool-calling workflows and agent patterns
- **Links to**: Specific tool call via `tool_call_id`

## Examples Overview

### 1. Simple Text Prompt (`01_model_simpleprompt.py`)
**Demonstrates**: Basic model invocation with a string prompt

- **Pattern**: Most straightforward approach
- **Use When**: Single, standalone request without conversation history
- **Input**: Plain string (auto-converted to HumanMessage internally)
- **Output**: AIMessage with response content
- **Example**:
  ```python
  response = model.invoke("write a haiku about the ocean")
  ```

### 2. Message Objects (`02_model_message.py`)
**Demonstrates**: Multi-turn conversations with message objects

- **Pattern**: Professional approach for maintaining conversation context
- **Use When**: Managing multi-turn conversations or complex interactions
- **Components**:
  - SystemMessage for instructions
  - HumanMessage for user queries
  - AIMessage for prior responses (to provide context)
- **Advantage**: Full type safety and IDE support
- **Example**:
  ```python
  messages = [
      SystemMessage("You are a poetry expert"),
      HumanMessage("Write a haiku about spring"),
      AIMessage("Cherry blossoms bloom...")
  ]
  response = model.invoke(messages)
  ```

### 3. Dictionary Format Messages (`03_model_message_dict_style.py`)
**Demonstrates**: OpenAI chat completions format for messages

- **Pattern**: Compatible with OpenAI API format
- **Use When**: Working with systems expecting standard API format
- **Format Specification**:
  ```python
  {
      "role": "system|user|assistant",
      "content": "message text"
  }
  ```
- **Note**: LangChain automatically converts to message objects internally
- **Recommendation**: Prefer message objects for better type safety
- **Example**:
  ```python
  messages = [
      {"role": "system", "content": "You are a poetry expert"},
      {"role": "user", "content": "Write a haiku about AI"}
  ]
  response = model.invoke(messages)
  ```

### 4. System Persona with Detailed Instructions (`04_model_message_system_persona.py`)
**Demonstrates**: Advanced system message with comprehensive persona definition

- **Pattern**: Best practice for consistent, high-quality responses
- **Use When**: You need specific model behavior, expertise, and response format
- **Best Practices**:
  - Define clear role and expertise
  - Specify expected output format
  - Include behavioral guidelines
  - Use multi-line messages for detailed instructions
- **Impact**: Significantly improves model consistency and response quality
- **Example**:
  ```python
  system_msg = SystemMessage("""
  You are a senior Python developer with expertise in web frameworks.
  Always provide code examples and explain your reasoning.
  Be concise but thorough in your explanations.
  """)
  ```

## Key Concepts

### Text Prompts vs Message Prompts

**Text Prompts** (strings):
- Simplest form for standalone requests
- No conversation history
- Minimal code complexity
- Example: `model.invoke("Your question here")`

**Message Prompts** (message objects):
- Full conversation management
- Multimodal content support
- System instructions
- Metadata tracking
- Recommended for production applications

### When to Use Each Pattern

| Scenario | Use |
|----------|-----|
| Single, standalone request | Text prompt |
| Multi-turn conversation | Message objects |
| Stateless API endpoint | Text prompt |
| Chatbot or conversational AI | Message objects |
| Need to track token usage | AIMessage with metadata |
| Tool calling/agents | ToolMessage + AIMessage |
| Need consistency in responses | System message with persona |

## LangChain Integration

All examples use `init_chat_model()` for model-agnostic initialization:
```python
from langchain.chat_models import init_chat_model

# Works with any supported provider (OpenAI, Anthropic, Ollama, etc.)
model = init_chat_model("ollama:model-name:tag")
```

## Message Metadata

Messages can include optional metadata:
```python
human_msg = HumanMessage(
    content="Hello!",
    name="alice",        # User identification
    id="msg_123"         # Unique identifier for tracing
)
```

AIMessages return usage metadata:
```python
response = model.invoke("Hello!")
response.usage_metadata  # {'input_tokens': 8, 'output_tokens': 304, ...}
```

## Advanced Usage

### Streaming Messages
For long responses, stream messages for incremental output:
```python
for chunk in model.stream(messages):
    print(chunk.text)
```

### Tool Calling
Messages support tool calls in AIMessage:
```python
response = model_with_tools.invoke("What's the weather?")
for tool_call in response.tool_calls:
    print(f"Tool: {tool_call['name']}")
    print(f"Args: {tool_call['args']}")
```
