# Generative UI with LangChain Agents - Complete Guide

## 🎨 What is Generative UI?

Generative UI is a paradigm where **AI agents generate complete user interfaces** as JSON specifications, rather than just returning text. Instead of the AI saying "here's the weather information," it generates an interactive weather card UI.

### Traditional vs. Generative UI

```
Traditional AI Chat:
User: "Create a login form"
AI: "Here's how to create one... [text instructions]"
Result: Static text response

Generative UI:
User: "Create a login form"
AI: generates → { "type": "Card", "children": ["TextInput", "TextInput", "Button"] }
Result: Interactive login form rendered instantly
```

## 📋 Key Concepts

### 1. **Component Catalog**
The "contract" between backend and frontend - defines what UI components are available.

```python
COMPONENT_CATALOG = {
    "components": {
        "Card": {
            "description": "A card container",
            "props": {
                "title": "string",
                "padding": "enum: sm|md|lg"
            }
        },
        "TextInput": {
            "description": "Text input field",
            "props": {
                "label": "string",
                "type": "enum: text|email|password"
            }
        },
        # ... more components
    }
}
```

### 2. **UI Specification**
JSON structure generated by the agent describing the UI tree.

```json
{
  "root": "login-form",
  "elements": {
    "login-form": {
      "type": "Card",
      "props": { "title": "Login" },
      "children": ["email-field", "password-field", "submit-btn"]
    },
    "email-field": {
      "type": "TextInput",
      "props": { "label": "Email", "type": "email" },
      "children": []
    },
    "password-field": {
      "type": "TextInput",
      "props": { "label": "Password", "type": "password" },
      "children": []
    },
    "submit-btn": {
      "type": "Button",
      "props": { "label": "Sign In", "variant": "primary" },
      "children": []
    }
  }
}
```

### 3. **Progressive Rendering**
As the agent streams the spec, elements arrive one at a time, and the UI builds up in real-time.

```
Time 1: Only root element → Shows Card with title
Time 2: Add TextInput → Shows Card with empty input
Time 3: Add Button → Shows Card with complete form
```

## 🛠️ How It Works: Step by Step

### Backend (Python with LangChain)

```python
1. Define Catalog
   └─ What components can agents use?

2. Create Agent
   └─ Pass catalog to system prompt
   └─ Agent knows about available components

3. User sends message
   └─ "Create a login form"

4. Agent generates JSON spec
   └─ Uses components from catalog
   └─ Spec is valid and renderable

5. Stream to frontend
   └─ Send spec as JSON
   └─ Or stream progressively
```

### Frontend (React)

```typescript
1. User types prompt
   └─ "Create a login form"

2. Send to agent backend
   └─ POST /agent/chat

3. Receive UI spec
   └─ Parse JSON

4. Render components
   └─ Use json-render library
   └─ Or custom rendering logic

5. Display to user
   └─ Interactive UI appears
   └─ User can interact with generated form
```

### API Endpoints

The backend provides these endpoints:

#### 1. **GET /health**
Health check
```bash
curl http://localhost:8000/health
```

#### 2. **GET /catalog**
Get available components
```bash
curl http://localhost:8000/catalog
```

#### 3. **POST /generate-ui**
Generate UI from description
```bash
curl -X POST http://localhost:8000/generate-ui \
  -H "Content-Type: application/json" \
  -d '{"description": "Create a login form"}'
```

#### 4. **POST /stream-ui**
Stream UI generation (Server-Sent Events)
```bash
curl -X POST http://localhost:8000/stream-ui \
  -H "Content-Type: application/json" \
  -d '{"description": "Create a login form"}'
```

#### 5. **POST /agent/chat**
Chat with agent using natural language
```bash
curl -X POST http://localhost:8000/agent/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Create a weather search widget"}'
```

## 🎯 Best Practices

### For Backend (Agent)

1. **Keep catalog focused**
   - Include only components needed for your use case
   - Too many components = worse AI decisions

2. **Write clear descriptions**
   - AI uses these to understand when to use components
   - "A vertical layout container" vs "layout thing"

3. **Validate specs**
   - Always check that generated specs are valid
   - Use TypeScript/Pydantic for type safety

4. **Provide good examples in system prompt**
   - "For a login form, use: Card > Stack > TextInput > Button"
   - Improves consistency

### For Frontend (UI)

1. **Progressive rendering**
   - Display UI as specs arrive, don't wait for complete
   - Pass `loading={true}` to renderer during streaming

2. **Error handling**
   - What if component doesn't exist?
   - What if props are invalid?
   - Provide fallbacks

3. **User feedback**
   - Show loading state while generating
   - Display errors clearly
   - Confirm successful rendering

## 🔗 Connecting Backend and Frontend

### Example: Full Integration

**Backend - Send UI Spec**
```python
@app.post("/agent/chat")
def agent_chat(request: dict):
    message = request.get("message")
    
    # Agent generates UI spec
    ui_spec = generate_ui_spec(message)
    
    return {
        "response": "I've created your form",
        "ui_spec": ui_spec.model_dump()
    }
```

```typescript

    body: JSON.stringify({ message })
  });

  // Render the spec
  setGeneratedSpec(data.ui_spec);
};
```

## 🚧 Advanced Topics

### Custom Components
Create your own components beyond the default catalog:

```python
COMPONENT_CATALOG = {
    "components": {
        "WeatherCard": {
            "description": "Displays weather information with icon and temp",
            "props": {
                "city": "string",
                "temperature": "number",
                "condition": "string",
                "icon": "string"
            }
        }
    }
}
```

### State Management
Components can have internal state:

```json
{
  "type": "TextInput",
  "props": { "value": "user input", "onChange": "handleChange" }
}
```

### Actions and Events
Wire up component interactions:

```python
"actions": {
    "handleSubmit": "calls /api/submit",
    "handleChange": "updates state"
}
```

## 🐛 Troubleshooting

### Common Issues

1. **Component not rendering**
   - Check if component is in catalog
   - Verify props match schema
   - Check console for errors

2. **Invalid JSON from agent**
   - Agent doesn't know about components
   - Improve system prompt with examples
   - Validate spec before rendering

3. **Performance issues**
   - Too many components = slow rendering
   - Large specs = slower streaming
   - Consider pagination/virtualization
